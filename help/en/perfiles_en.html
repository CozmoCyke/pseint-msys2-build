Language Customization

There are several pseudocode features that can be customized to adapt PSeInt to each teacher's preferences. By default, PSeInt uses the most flexible and natural configuration possible, but it also offers a list of predefined configurations called "profiles". The profile list can be opened from the item "Language Options..." in the "Configure" menu.

If none of the profiles is suitable, you can use the "Customize" button located below the profile list to define each of the following customizations one by one:

  • Use of uninitialized variables: if this option is disabled, variables that were never read or assigned may still be used in expressions or output. In that case they assume the default value 0, "", or FALSO depending on the corresponding type. If the option is enabled, writing or evaluating an expression that involves an uninitialized variable generates a runtime error.

  • Mandatory explicit definition: if this option is enabled, variables cannot be used until their type has been defined with the keyword DEFINIR.

  • Mandatory semicolon: if this option is enabled, sequential instructions must end with a semicolon (;), as in languages such as C++ or Pascal. If it is disabled, the semicolon is optional.

  • String concatenation with +: if this option is enabled, the content of two character variables can be concatenated with the + operator.

  • String functions: enables Longitud, Subcadena, Mayúsculas, Minúsculas, and Concatenar.

  • Word operators: enables Y, O, NO, and MOD as synonyms for &, |, ~, and %.

  • Zero-based arrays: when enabled, the first element of an array of N elements is 0 and the last is N-1; otherwise the first is 1 and the last is N.

  • Variable expressions in dimensions: allows variables or expressions to be used when dimensioning arrays.

  • Assignment with =: enables the equal sign as an assignment operator.

  • Functions/SubProcesses: enables the keywords SubProceso and Función to define new functions in pseudocode.

  • Flexible syntax: enables optional syntax variants such as omitting HACER in some loops, using DESDE in FOR, separating READ/WRITE expressions with spaces, and other more colloquial constructions.

  • Colloquial conditions: allows conditions such as "X ES PAR" or "X NO ES MULTIPLO DE 5".

  • Numeric-only SEGUN selector: if enabled, the multiple-selection structure only accepts integer control expressions; otherwise character variables may also be used.

  • Automatic default FOR step: if enabled, when the step is omitted it uses +1 or -1 according to the initial and final values. If disabled, the default step is always +1.

  • Nassi-Schneiderman diagrams: controls whether that diagram style is used instead of classic flowcharts.

  • Alternative input/output symbols: changes the geometric shapes used for input and output actions in the classic flowchart editor.

  • Accents in identifiers: allows accents, diaereses, and the Spanish ñ in variable and function names.

  • Preferred algorithm keyword: selects whether generated code uses ALGORITMO or PROCESO.

  • Preferred function keyword: selects whether generated code prefers FUNCIÓN rather than the other equivalent keywords.

  • Allow Repetir ... Mientras Que: enables the use of "Mientras que <condición>" instead of "Hasta que <condición>" to close a Repeat structure.

  • Prefer Repetir ... Mientras Que: when enabled, the command panel inserts the true-iterating Repeat variant by default.


Below is an example pseudocode written for the Strict and Flexible profiles, illustrating some of the differences between configurations:

Proceso EjemploEstricto
   Definir VAL_MAYOR,ARREGLO,CANT,I Como Enteros;
   Dimension ARREGLO[10];
   Escribir "Ingrese la cantidad de números:";
   Leer CANT;
FinProceso
   Algoritmo EjemploFlexible
   Escribir "Ingrese la cantidad de números:"
   Leer CANT
   Dimension ARREGLO[CANT]
FinAlgoritmo
