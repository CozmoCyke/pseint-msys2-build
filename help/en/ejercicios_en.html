Self-contained Exercises

A self-contained exercise is a special file that, when opened with PSeInt, presents a problem to be solved by completing an algorithm and offers a button to evaluate the solution automatically and determine whether it produces the correct result. These files are identified by the extensions .psz and .psx. They are loaded in the editor like any other pseudocode file, using the "Open" option in the File menu or by dragging the file from a file explorer into the PSeInt window. If the file has the .psx extension, a password is requested before it can be opened.

When a self-contained exercise is opened, PSeInt shows a small panel at the bottom of the window with the "Evaluate" button and, depending on the exercise, it may also show a statement or description of the problem to solve in the quick-help panel and an initial pseudocode base to work from.

The student can use all the other PSeInt features to solve the problem, such as the execution terminal, step-by-step execution, the flowchart editor, and so on. Once the exercise is solved, clicking the "Evaluate" button runs the algorithm and verifies the output it writes on screen for a predefined set of inputs.

The input set (test cases) is stored in the exercise file. During evaluation, the student does not see the normal execution terminal, only a progress bar. After execution, a message defined by the exercise author reports the result. If one or more test cases fail, PSeInt may show the inputs corresponding to those cases and optionally the correct outputs, if the exercise is configured to allow it.

You can find a sample exercise in the "ejemplos" folder of your PSeInt installation. Click here to open the example.

For information about how to create a self-contained exercise file, see the teacher-information section Exercise Creation.
