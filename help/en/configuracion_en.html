Configuration

Appearance:
  • Fonts: You can configure the fonts used for pseudocode and for the execution terminal with the Select fonts... option in the Presentation submenu of the Configure menu. You can also temporarily change the font size (zoom) in a single pseudocode or terminal tab by holding Control and rotating the mouse wheel.
  • Colors while running: If you enable the option to use colors while interpreting, the interpreter uses different colors for each event type. Errors are shown in red, output messages in green, input data in yellow, and additional information in blue.


Dockable Panels:
  • Variables panel: shows the variables identified in an algorithm, organized by process and subprocess. The icon represents the data type. This information is extracted before execution, so some types may not be fully determined yet and some arrays may not have fully defined dimensions.
  • Operators panel: presents a catalog of predefined functions and constants in PSeInt together with the list of possible operators, organized by categories. Clicking one inserts it into the pseudocode at the current cursor position.
  • Command panel: lets you insert actions and control structures into your algorithm with a click. It is normally shown on the left side of the window and inserts templates with highlighted placeholders that you must complete.
  • Step-by-step panel: the step-by-step execution panel allows more detailed control of algorithm execution and also configures desk checks.
  • Quick help panel: helps complete templates from the command panel and provides details and suggestions to correct interpreter errors. It is shown automatically at the bottom of the window in the corresponding situations.


Writing Assistance:
  • Autocomplete: shows keyword suggestions while you type and lets you select them so you do not have to type the whole word.
  • Automatic closing of structures: when enabled, pressing Enter after starting a control structure adds the corresponding closing keyword if needed.
  • Structure highlighting: when the cursor is placed on the first or last line of a control structure, the related lines are highlighted to show its scope.
  • Calltips: indicate what should be written after each keyword. They are shown as floating tips over the code and describe what kind of action, expression, or value should be entered next.
  • Smart indentation: adds or removes indentation so the code is aligned correctly according to its structure and nesting level.
  • Check syntax while typing: verifies the pseudocode automatically a few seconds after the user stops typing. Errors are underlined in red and can show a short description when the cursor is placed over them.


Language:
  • PSeInt allows some aspects of the pseudo-language to be configured to make it stricter or more permissive, or to adapt it to a teacher's needs. For that purpose it provides configuration profiles. Teachers may send the profile they want to use together with the data for their course and institution so that it appears in the list.
  • This configuration is performed the first time PSeInt is run, but it can be modified later from the corresponding item in the Configure menu. If none of the available profiles is suitable, the selection dialog offers a "Customize" button that lets you set each language option individually.
