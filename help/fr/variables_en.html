Variables

    A variable in a computational algorithm is a memory position where information can be stored. For example, if a program must obtain the area of a triangle, its base and height will probably be loaded into memory in two variables so the calculation can be performed. The result will probably also be assigned to a variable after the calculation so it can then be shown to the user. As its name suggests, the value stored in a variable can change as the program progresses. In pseudocode the concept is similar. A variable represents a place where certain information can be stored.

    In an algorithm or program, a variable is referenced through an identifier (the variable name). An identifier must begin with letters, and can contain only letters, numbers, and the underscore. It cannot contain spaces or operators, and it cannot match a reserved word or language function, in order to avoid ambiguity. Examples of valid identifiers are: A, B, C, Side1, Total, Name_and_Surname, EmailAddress, ... In most real languages variable names cannot contain accents, diaereses, or the Spanish letter n with tilde. In PSeInt this is allowed, depending on the selected language profile.

    In PSeInt variables have an associated data type, so during the execution of the algorithm a variable must always store data of the same type. For example, if a variable is used to store numbers, it cannot later be used to store text. You can explicitly declare the type of a variable with the keyword Define, or you can let the interpreter try to infer it from the data stored in it and from the way it is used in the algorithm. If you use the default language profile (Flexible), explicit definition is optional, but the language can be configured to require it.

    There are two ways to create a variable and/or assign it a value: input and assignment. If a value is read into or assigned to a variable that does not exist, it is created. If the variable already existed, it takes the new value and loses the old one. For this reason, assignment and input are said to be destructive actions (although it should be noted that in an assignment more than one variable may take part, and only the previous contents of the variable to the left of the assignment sign are destroyed). Once initialized, the variable can be used in any expression (to perform a calculation in an assignment, to display something on screen, as a condition in a control structure, etc.)
