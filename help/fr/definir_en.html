Explicit Definition

   The Define instruction makes the type of one or more variables explicit. This definition may be optional or mandatory depending on the language configuration. The syntax is:

      Definir <var1> , <var2> , ... , <varN>  Como [REAL/ENTERO/LOGICO/CARACTER];

    A variable must be defined before it is used for the first time. Arrays are defined by using their identifier (without indices or dimensions) before or after dimensioning them, and the type applies to all their elements because they are homogeneous data structures.

    The possible types are NUMERO, NUMERICO, REAL, ENTERO, LOGICO, CARACTER, TEXTO, and CADENA.

    NUMERO, NUMERICO, and REAL are synonyms for the basic numeric type, which can store both real numbers and integers. The ENTERO type is a specialization that stores only integer values; any non-integer value that is read or assigned to a variable of this type is truncated.

    A variable of type LOGICO can only take the values VERDADERO and FALSO, but when a variable already defined as logical is read, the user may also enter the abbreviations V and F, or 0 and 1.

    CARACTER, TEXTO, and CADENA are synonyms for defining string variables. They can contain zero, one, or more arbitrary characters and have no maximum length. If a variable of this type is declared and the user enters a number or a logical value in an input instruction, a string containing that text is assigned instead.

    If you try to assign data of the wrong type to a variable that is already defined, a runtime error occurs.

   When the language configuration enables flexible syntax, variable types can also be defined in a more colloquial alternative form using the keywords Es and Son, as follows:

      <var> Es [REAL/ENTERO/LOGICA/CARACTER];
      <var1> , <var2> , ... , <varN>  Son [REALES/ENTEROS/LOGICAS/CARACTERES];
