PSeudoCode

   The characteristics of this pseudo-language were proposed in 2001 by the professor responsible for the Programming Fundamentals course (Horacio Loyarte) in the Computer Engineering program at FICH-UNL. Its premises are:
  • Simple syntax
  • Handling of the basic control structures
  • Only 3 basic data types: numeric, character / strings, and logical (true-false).
  • Data structures: arrays


More details:
  • General form of an algorithm in pseudocode
  • Use of variables in pseudocode
  • Simple data types (logical, numeric, character) and arrays
  • Building expressions (functions and operators)
  • Sequential primitives (input, output, assignment, others)
  • Conditional and looping control structures
  • Creating SubProcesses/Functions
  • Other considerations
  • Language customization
