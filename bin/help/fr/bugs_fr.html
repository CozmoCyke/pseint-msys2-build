Bug Report

Things to keep in mind to report a bug in this software correctly:
  • The proper place to do so is the bug report forum on the PSeInt site.
  • Make sure you are using the latest version.
  • Describe the problem as completely and in as much detail as possible. Copy the error messages exactly, include the output you expected and the output you actually obtained, and so on.
  • Explain how to reproduce it: the steps that lead you to the problem, example pseudocode, the values you enter while running it, and so on.
  • Indicate which version of PSeInt and which operating system you are using (you can check the "About..." item in the "Help" menu).
  • Check the forum regularly because more information may be needed to fix the problem, and so you can know when a solution is available.
  • Look to see whether the bug already has an open topic in the forum. If it does, add your data to that same topic. If not, start a new one.
  • If it is NOT a problem in the interpretation or translation logic, use this link to restart PSeInt in a special mode that generates a text file with information that may be useful for analyzing the issue. Attach the generated file to the bug report.
  • Finally, remember that the author maintains this software in his spare time and receives many reports and suggestions every week, so requests must be prioritized, answers may take time, and it will not always be possible to fix every issue.
